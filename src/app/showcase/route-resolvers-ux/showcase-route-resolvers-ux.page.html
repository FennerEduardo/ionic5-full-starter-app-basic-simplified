Lets see the difference between using hot-observables and cold-observables for Route Resolves

By default, Angular Route Resolvers won't transition to the page until the Resolver Observable completes.

Let's suppose the backend is slow and takes 5 seconds to fetch data and return it to the client, this is what's going to happen.

Expected behavior: Page transition will be delayed 5 seconds until the server sends data back to the client.

Go to Cold Observable Resolver Page

By using the SubjectFetch utility in conjuction with Angular Route Resolvers, we can solve this UX problem and make page transitions inmediate.

Expected behavior: Page transition will be instant and the transitioned page will show some shell elements while the server sends data back to the client.
